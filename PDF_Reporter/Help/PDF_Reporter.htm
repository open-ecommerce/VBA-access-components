PDF Reporting Component

 

Introduction:

More than a component this is an example of how to use the dynapdf.dll PDF converter Library. 

 

You will need to have installed the Snapshot Viewer for Microsoft Access before you open the database in order to work:

http://www.microsoft.com/en-us/download/details.aspx?id=13911

 

(also check the rest of the dependencies)

 

 

This application also explains how to use the following tools:

  • Snapshoot Viewer to incorporate Access reports into a form
  • Create customable tool tips with the clsToolTip class.
  • Drag & Drop List boxes.
  • Export Access reports as xls, snp, pdf files and send them as email attachments.

 

 

 

Parts of the Component:

External References Required:

The dynapdf.dll and StrStorage.dll have to be in the same folder of the application or into the System32 folder there is no need to register.

 

The Snapshot Viewer Control has to be also added to the database.

 

 

How to use it:

Because the reports are related to specific fields the form has to be rebuilding depending of the different filters wanted for the report.

 

The system reports can be grouped in areas into the tblAuxReports and the details for each report are populated into tblAuxReportsDetails.

 

The proCreateQuery and funBuildConditions functions, builds a new reports query based on the provided query for the area, the fields and the filter options selected by the user.

 

 

Adding a new report based on an existing area:

 

  1. Create the report base on a query, trying to use the one all ready existing for this area.

In the demo is qry_repContacts.

  1. Add a new record in tblAuxReportsDetails with the name of the report, the query used and the fields you want to order by and display in the table.

Adding new conditions to the query:

 

  1. Add the controls needed for the new conditions.

 

  1. Add a new case statement to the funBuildConditions function reflecting the new conditions.
    If the filter was made in a 3 states combo box the statement can be some thing like that:

 

    'Countries

    Select Case Nz(cboCountries, 0)

        Case 0

        Case c_MultipleSelections

            If cboCountries.Tag <> "" Then

                strList = Replace(cboCountries.Tag, "|", ",", , , vbTextCompare)

                strCriteria = strCriteria & "IdCountries IN(" & strList & ")"

            End If

        Case Else

            strCriteria = strCriteria & "IdCountries = " & cboCountries.Value

    End Select

 

 

 

Limitations:

  • Because of the nature of component, the users criteria have to be developed depending on the report.  Maybe a solution could be building the filter controls dynamically on an external form based on a table.
